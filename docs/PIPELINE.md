# 콘텐츠 제작 파이프라인

> 아이디어부터 인스타그램 게시, 회고까지의 전체 워크플로우

## 전체 흐름

```
[1] 아이디어 캡처   →  content/ideas/*.md
       ↓
[2] 콘텐츠 기획     →  content/drafts/*.md  (템플릿 채우기)
       ↓
[3] 본문 작성       →  카드별 텍스트 + 카피 확정
       ↓
[4] 비주얼 제작     →  AI 도구로 이미지/배경/카드 생성
       ↓
[5] 카드뉴스 조립   →  Canva / 미리캔버스 / 코드 자동화
       ↓
[6] 캡션 + 해시태그 →  templates/instagram-caption/
       ↓
[7] 검토 (자가)     →  체크리스트 (오타, 추천도, 출처)
       ↓
[8] 게시 / 예약     →  Meta Business Suite or Buffer
       ↓
[9] 회고            →  reviews/YYYY-MM-DD-주제.md
       ↓
[10] 아카이브       →  content/published/
```

---

## 단계별 상세

### [1] 아이디어 캡처 — 5분 이하

**언제**: 방탈출 끝나고 나오자마자, 다른 사람 후기 보다가, 새 매장 발견했을 때.

**어디에**: `content/ideas/YYYY-MM-DD-짧은제목.md`

**최소 필드**:
```markdown
---
type: recommendation | course | news | review
created: 2026-05-25
status: idea
---

# 한 줄 아이디어
(예: 강남 신논현 4시간 코스 - X카페→A테마→B테마→식사→C테마)

## 왜 이게 좋은 콘텐츠인가
- (예: 동선이 짧고 난이도가 자연스럽게 올라감)

## 필요한 추가 정보
- [ ] 각 테마 사진
- [ ] 시간/가격 재확인
- [ ] 친구 의견 1명
```

> **개발자 팁**: 모바일에서 메모만 빠르게 → 데스크탑에서 마크다운으로 옮기기. iOS 메모/노션/옵시디언 어디든 OK, 결국 `ideas/` 로 흘러들어오면 됨.

### [2] 콘텐츠 기획 — 15~30분

아이디어 중 게시할 것을 선택 → 해당 타입의 템플릿 복사:

```bash
cp templates/card-news/review.md content/drafts/2026-05-25-제로월드강남-비밀의화원.md
```

템플릿이 카드별 슬롯(표지/본문 1~6/마무리)을 강제하므로 디자인 결정 줄임.

---

### [3] 본문 작성 — 30~60분

**한 카드 = 한 메시지** 원칙.
- 표지: 후킹 (질문, 숫자, 강한 단어)
- 본문 카드: 한 카드당 글자 수 50~120자 (이미지 위에 얹힐 분량)
- 마무리: CTA (저장 유도, 댓글 유도, 다음편 예고)

**카피 작성 시 AI 활용** (자세한 도구는 [AI_TOOLS.md](AI_TOOLS.md)):
```
Claude/ChatGPT 프롬프트 예시:
"방탈출 후기 인스타그램 카드뉴스용 표지 문구 5개 후보를 줘.
테마: 제로월드 강남 '비밀의 화원', 난이도 4/5, 추천도 5/5,
포인트는 '예상 못한 페이크 트릭'. 톤: 약간 호기심 자극, 스포 없이."
```

---

### [4] 비주얼 제작 — 30~90분

3가지 경로 중 택1 ([AI_TOOLS.md](AI_TOOLS.md) 참고):

| 경로 | 도구 | 시간 | 퀄리티 |
|------|------|------|--------|
| **A. 직접 촬영 + 보정** | iPhone + Lightroom | 빠름 | 진정성 ↑ |
| **B. AI 생성** | Midjourney/Ideogram/DALL-E | 중간 | 분위기 ↑ |
| **C. 스톡 + 합성** | Unsplash + Canva | 빠름 | 무난 |

> 일관성을 위해 **컬러 팔레트**와 **폰트 1~2개**를 [BRAND_GUIDE.md](BRAND_GUIDE.md) 에 고정.

---

### [5] 카드뉴스 조립 — 30~60분

> **캔버스 사이즈**: 1080×1350 (4:5 세로형) — 자세한 이유는 [BRAND_GUIDE.md](BRAND_GUIDE.md#카드뉴스-캔버스-사이즈) 참고. 캐러셀 내 모든 슬라이드는 같은 사이즈여야 함.

**옵션 1: 디자인 도구 (반자동, 추천)**
- 미리캔버스 → 인스타그램 → **세로형 이미지** 선택 (1080×1350)
- 자체 템플릿 만들고 텍스트만 교체
- 한 번 만들면 다음부터는 10분 컷

**옵션 2: 코드 자동화 (서버 개발자 친화)**
- HTML/CSS 템플릿 + Puppeteer로 PNG 렌더링
- Bannerbear / Placid API로 JSON → 이미지
- 자세한 구현은 [pipeline/README.md](../pipeline/) 참고

```javascript
// 예시: pipeline/render-card.js (의사 코드)
const data = require('../content/drafts/2026-05-25-foo.md');
await bannerbear.createImage({
  template: 'review-cover',
  modifications: [
    { name: 'title', text: data.title },
    { name: 'rating', text: '⭐'.repeat(data.rating) },
  ],
});
```

### [6] 캡션 + 해시태그 — 10분

`templates/instagram-caption/` 의 타입별 템플릿 사용.

**해시태그 전략**:
- 대형 (10만+): #방탈출 #방탈출추천 #방탈출후기
- 중형 (1~10만): #강남방탈출 #서울방탈출 #방탈출카페
- 소형 (1만 미만, 발견되기 쉬움): #방탈출일정 #방탈출코스 #방탈출덕후
- 브랜드: #jroom_official #제이룸방탈출 (계정 핸들 `@jroom.official` 기반)

---

### [7] 검토 체크리스트

게시 전 다음을 확인:

- [ ] 오탈자 (마지막에 소리내어 읽기)
- [ ] 카드 순서가 자연스러운가
- [ ] 첫 카드만 봐도 클릭하고 싶은가
- [ ] 마지막 카드에 CTA 있는가
- [ ] 매장명/테마명 정확한가
- [ ] 스포일러 없는가
- [ ] 추천도/난이도 일관성 있는가 (다른 후기들과 비교)
- [ ] 캡션이 250자 이내 (모바일 첫 화면)
- [ ] 해시태그 20~30개 (대/중/소 섞임)

---

### [8] 게시 / 예약

**즉시 게시**: 인스타그램 앱
**예약 게시**: Meta Business Suite (무료) / Buffer / Later

> **방탈출 콘텐츠 최적 시간대 (추정)**: 평일 19~21시, 주말 11~13시 (방탈출 다녀온 직후 검색하는 시간)

---

### [9] 회고 — 게시 3~7일 후

`reviews/YYYY-MM-DD-주제.md` 생성:

```markdown
---
post_date: 2026-05-25
type: review
---

# 성과
- 좋아요: 152
- 저장: 38
- 도달: 4,210
- 팔로워 증감: +12

# 잘 된 점
- 표지의 "절대 풀 수 없다고 했던 그 자물쇠" 가 후킹 잘됨

# 아쉬운 점
- 4번 카드 글자가 너무 작았음 (모바일에서 안 읽힘)

# 다음에 적용할 것
- 카드 글자 최소 36pt로 강제
```

---

### [10] 아카이브

`content/drafts/2026-05-25-foo.md` → `content/published/2026-05-25-foo.md` 로 이동.
Git에 커밋: `git commit -m "publish: 제로월드 강남 비밀의 화원 후기"`

## 주간 운영 루틴 (제안)

| 요일 | 작업 |
|------|------|
| **월** | 아이디어 정리 + 이번주 게시 3개 선정 |
| **화** | 1번 콘텐츠 본문 작성 + 카드뉴스 |
| **수** | 1번 게시 |
| **목** | 2번 콘텐츠 본문 작성 + 카드뉴스 |
| **금** | 2번 게시 |
| **토** | 방탈출 다녀오기 (자료 수집) |
| **일** | 3번 콘텐츠 + 회고 작성 |

> 처음에는 주 1회로 시작 → 익숙해지면 주 2~3회로 확장.

---

## 자동화 우선순위 (개발자 시점)

1. **MVP (수동 + AI 도구)**: Canva 템플릿 + Claude로 카피 → 직접 게시
2. **반자동**: 마크다운 파일 → 스크립트가 JSON 변환 → Bannerbear API 호출
3. **완전 자동**: 위 + Meta Graph API로 예약 게시 + Notion DB와 양방향 동기화

각 단계는 [pipeline/](../pipeline/) 에서 점진적으로 구축.
