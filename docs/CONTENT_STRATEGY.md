# 콘텐츠 전략 — 생산은 몰아서, 발행은 규칙적으로

> 핵심 원칙: **생산 캘린더(만드는 날)** 와 **발행 캘린더(올리는 날)** 를 분리한다.
> 영감 있을 때 몰아서 만들어 재고를 쌓고, 발행은 정해진 요일에 기계적으로.

---

## 1. 운영 리듬

### 발행 (고정)
- **주 2회**: **수요일 20시 / 토요일 11시**
- 수요일 = 평일 저녁(퇴근 후 방탈출 검색 시간)
- 토요일 = 주말 오전(주말 약속 짤 때)
- 월 8개 발행

> 시간대는 인사이트 데이터 쌓이면 조정. 일단 이걸로 시작.

### 생산 (유동)
- 시간/영감 있을 때 **몰아서 2~5개씩** 제작
- 목표: 항상 **2주치(4개) 이상 재고** 유지
- 재고가 2개 이하로 떨어지면 → 다음 생산 세션 우선순위 ↑

### 재고 상태 추적
`content/ideas/BACKLOG.md` 의 상단에 현재 재고(완성 대기 중) 표시:
```
🟢 재고: 5개 (2.5주분) — 여유
🟡 재고: 2~3개 — 곧 생산
🔴 재고: 0~1개 — 긴급 생산
```

---

## 2. 발행 캘린더 (4주 로테이션)

**수요일 슬롯 = 후기 고정** (가장 꾸준히 생산 가능, 계정 정체성)
**토요일 슬롯 = 로테이션** (다양성 확보)

| 주차 | 수(20시) | 토(11시) |
|------|---------|---------|
| 1주 | 후기 | 테마별 추천 |
| 2주 | 후기 | 연방 코스 |
| 3주 | 후기 | 입문/꿀팁 |
| 4주 | 후기 | 랭킹/비교 (또는 시즌 특집) |

→ 한 달에 후기 4개 + 추천/코스/꿀팁/랭킹 각 1개. 안정적이면서 다양함.

> **후기 재고가 부족한 주**: 토요일 슬롯 포맷을 수요일로 당기고, 토요일은 가벼운 밈/Q&A로 대체. 공백보다 낫다.

## 3. 콘텐츠 카테고리 맵

### 🎯 코어 3종 (계정의 기둥 — 매주 발행)

#### A. 방탈출 후기 (review) ⭐ 주력
직접 체험한 1개 테마 솔직 리뷰. 별점·추천대상·정보.
- **강점**: 본인 경험 = 무한 소재, 진정성, 신뢰도
- **빈도**: 주 1회 (수요일 고정)
- 템플릿: `templates/card-news/review.md`

#### B. 연방 코스 (course)
위치 기반 연속 방탈출 타임라인 + 동선 + 테마 소개.
- **강점**: "저장" 많이 받는 실용 콘텐츠 (알고리즘 ↑)
- **빈도**: 월 1회
- 템플릿: `templates/card-news/course.md`

#### C. 테마별 추천 (recommendation)
입문/공포/추리/4인 등 큐레이션 TOP 5.
- **강점**: 공유·저장 유도, 신규 유입
- **빈도**: 월 1회
- 템플릿: `templates/card-news/recommendation.md`

---

### 🌟 확장 포맷 (다양성·도달 확대 — 로테이션/특집)

#### D. 입문 가이드 / 꿀팁 (guide)
방탈출 처음인 사람을 위한 정보성 콘텐츠.
- 예: "방탈출 처음이세요? 알아두면 좋은 7가지"
- 예: "힌트 잘 쓰는 법 / 안 쓰고 깨는 법"
- 예: "방탈출 용어 사전 (장치/자물쇠/페이크 등)"
- 예: "방탈출 매너 — 이건 하지 마세요"
- **강점**: 저장 폭발, 입문자 팔로우 전환율 ↑

#### E. 랭킹 / 베스트 (ranking)
- 예: "2026 상반기 내 인생테마 BEST 5"
- 예: "올해 가장 무서웠던 테마 TOP 3"
- 예: "재방문하고 싶은 테마"
- **강점**: 논쟁·댓글 유도 (덕후들 의견 충돌 = 참여 ↑)

#### F. 비교 (comparison)
- 예: "키이스케이프 vs 제로월드, 뭐가 다를까"
- 예: "같은 공포인데 A테마 vs B테마"
- 예: "프랜차이즈별 난이도 체감 비교"
- **강점**: 정보 가치 ↑, 검색 유입

#### G. 공감 / 밈 (meme)
- 예: "방탈출러 공감 — 이런 적 있죠?"
- 예: "방탈출 같이 가면 안 되는 친구 유형"
- 예: "탈출 실패했을 때 우리 표정.jpg"
- **강점**: 도달·공유 폭발, 부담 적은 제작 (가끔 양념용)

#### H. Q&A / 자주 묻는 질문 (qna)
- 예: "방탈출 자주 묻는 질문 5개 답해드림"
- 예: "DM으로 가장 많이 온 질문"
- **강점**: 상호작용, 팔로워와 친밀감

#### I. 신규/소식 (news)
- 예: "이번 달 새로 오픈한 테마 모음"
- 예: "예약 전쟁 — 오픈런 해야 하는 테마"
- **강점**: 시의성, 정기 코너화 가능

#### J. 시즌 특집 (seasonal)
- 예: "할로윈에 갈 공포 테마", "크리스마스 감성 테마"
- 예: "연말 방탈출 결산"
- **강점**: 시즌 검색 트래픽

## 4. 카테고리별 비중 가이드 (월 8개 기준)

| 카테고리 | 월 개수 | 비고 |
|---------|--------|------|
| 후기 (A) | 4 | 주력, 수요일 고정 |
| 테마별 추천 (C) | 1 | |
| 연방 코스 (B) | 1 | |
| 꿀팁/랭킹/비교 중 (D/E/F) | 1~2 | 로테이션 |
| 밈/Q&A/소식 (G/H/I) | 0~1 | 양념, 여유될 때 |

> 황금비율: **정보(후기·추천·코스) 70% + 가벼운 콘텐츠(밈·Q&A) 30%.**
> 정보만 있으면 딱딱하고, 밈만 있으면 신뢰 안 쌓임.

## 5. 배치 생산 워크플로우

영감/시간 있는 날 한 번에:

```
[생산 세션 — 2~3시간]
1. BACKLOG.md 에서 만들 주제 3~5개 선택
2. 각 주제 → content/drafts/ 에 템플릿 복사 후 텍스트만 채우기 (카피)
   (디자인은 아직 X, 텍스트만 — 머리 한 모드로 몰아서)
3. 카피 다 되면 → 미리캔버스에서 한 번에 디자인 (손 한 모드로 몰아서)
4. 완성 PNG → 각 draft 폴더 또는 파일명 규칙으로 저장
5. 상태를 "ready"로 표시 (frontmatter status: ready)
```

> **핵심**: "글쓰기 모드"와 "디자인 모드"를 섞지 말 것. 뇌 전환 비용이 큼. 카피 5개 몰아 쓰고 → 디자인 5개 몰아 하기.

## 6. 발행 워크플로우 (발행일에)

```
[발행 — 10분]
1. content/drafts/ 에서 status: ready 인 것 중 캘린더에 맞는 카테고리 선택
2. 미리 만든 PNG + 캡션을 Meta Business Suite(또는 Buffer)에 업로드
3. 캘린더 시간에 예약 (또는 즉시 게시)
4. 파일을 content/published/ 로 이동 + git commit
```

> 더 나아가면: 한 달치를 한 번에 예약 게시 걸어두기 (Buffer/Meta Business Suite) → 발행 자체를 자동화.

---

## 7. 상태(status) 라이프사이클

frontmatter의 `status` 필드로 관리:

```
idea     → 백로그에만 있는 주제 (아직 안 만듦)
draft    → 카피 작성 중
ready    → 카피+디자인 완성, 발행 대기 (= 재고)
scheduled→ 예약 게시 걸어둠
published→ 게시 완료
```

재고 = `status: ready` 개수. 이게 4개 이상이면 안심.
