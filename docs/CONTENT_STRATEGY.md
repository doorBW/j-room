# 콘텐츠 전략 — 생산은 몰아서, 발행은 규칙적으로

> 핵심 원칙: **생산 캘린더(만드는 날)** 와 **발행 캘린더(올리는 날)** 를 분리한다.
> 영감/시간 있을 때 몰아서 만들어 재고를 쌓고, 발행은 정해진 요일에 기계적으로.

---

## 1. 콘텐츠 카테고리 (4종)

| 카테고리 | 성격 | 생산 난이도 | 역할 |
|---------|------|-----------|------|
| **장르/주제별 추천** (recommendation) | 메인 ⭐ | 낮음 (경험 풀에서 추출) | 주력·가장 안 끊김 |
| **연방 코스** (course) | 메인 | 높음 (여러 테마 정보 필요) | 저장·실용, 차별화 |
| **방탈출 소식** (news) | 정기 | 낮~중 (AI 집계) | 시의성·정기 코너 |
| **방탈출 후기** (review) | 비정기 | 중 (직접 다녀와야) | 진정성·신뢰 |

> 확장 포맷(꿀팁/랭킹/비교/밈/Q&A 등)은 계정이 자리잡은 뒤 추가 검토. 지금은 4종에 집중.

### 카테고리 상세
- **추천**: 장르(공포/추리/문제풀이/판타지/코믹)·주제(입문용/지역/인원수)별 방탈출 2~5개 큐레이션. → [CONTENT_TYPES.md](CONTENT_TYPES.md)
- **연방 코스**: 위치 기반 연속 방탈출 타임라인 + 동선 + 테마 소개. 본인이 순서·시간 제공.
- **소식**: 신규 오픈/신규 테마/이벤트/예약 소식을 주간 집계 → 카드뉴스. (AI 활용, 아래 §5)
- **후기**: 직접 체험한 1개 테마 솔직 리뷰. 당분간은 과거 경험 회상으로 생산.

## 2. 운영 리듬

### 발행 (고정 + 보너스)
- **기본 주 2회**: **수요일 20시 / 토요일 11시**
- **보너스 주 1회**: **일요일 20시** — 재고(추가 생산분) 있을 때만
- 수 = 평일 저녁(퇴근 후 검색), 토 = 주말 오전(약속 짤 때), 일 = 주말 마무리

> 시간대는 인사이트 데이터 쌓이면 조정. 일단 이걸로 시작.

### 생산 (유동)
- 시간/영감 있을 때 **몰아서 2~5개씩** 제작
- 목표: 항상 **2주치(기본 4개) 이상 재고** 유지
- 재고가 2개 이하로 떨어지면 → 다음 생산 세션 우선순위 ↑

## 3. 발행 캘린더 (2주 격주 사이클)

```
        수 20시      토 11시      일 20시(보너스)
1주차 │  추천    │   연방    │   후기 (생산 시)
2주차 │  소식    │   후기    │   추천 (생산 시)
```

**기본 발행(보너스 제외) 월간 환산** (월 ≈ 4.3주 = 2.17 사이클):
| 카테고리 | 월 발행 (기본) | 보너스 포함 시 |
|---------|--------------|--------------|
| 추천 | ~2회 | ~4회 |
| 연방 | ~2회 | ~2회 |
| 소식 | ~2회 | ~2회 |
| 후기 | ~2회 | ~4회 |

### 유연성 규칙 (재고 없을 때)
- **후기 재고 없는 주** → 그 슬롯을 **추천**으로 대체 (추천은 항상 생산 가능 = 만능 백업)
- **연방 재고 없는 주** → 추천 또는 소식으로 대체
- **보너스(일) 슬롯**: 재고 없으면 그냥 건너뜀. 공백 OK. (기본 2회는 사수)

> 원칙: **빈 슬롯을 억지로 채우려 무리한 콘텐츠 올리지 말 것.** 추천으로 메우거나 건너뛰기. 일관성 > 빈도.

## 4. 카테고리 비중 (기본 발행 기준, 2주 = 4개)

```
1주차: 추천 ■  연방 ■
2주차: 소식 ■  후기 ■
```
→ 4종이 2주에 골고루 1번씩. 보너스가 붙으면 추천·후기가 가산.

황금비율: **메인 콘텐츠(추천·연방·후기) + 시의성(소식)** 의 균형. 4종 체제 자체가 이미 균형 잡힘.

## 5. 소식(news) 카테고리 — AI 집계 워크플로우

> 목표: 주 1회(2주차 수요일), 최근 방탈출 소식을 모아 카드뉴스로.
> 지금은 반자동으로 시작 → 점차 자동화.

### Phase 1: 수동 (지금 시작 가능)
```
1. 소스 수집 (주 1회 30분):
   - 인스타 해시태그 (#방탈출신규, #방탈출오픈, 매장 공식계정)
   - 방탈출 커뮤니티 (네이버 카페, 디스코드)
   - 주요 매장 공식 SNS / 예약 페이지
2. 신규 오픈 / 신규 테마 / 이벤트 / 예약 오픈 3~5건 추리기
3. AI(Claude/ChatGPT)로 카드뉴스 카피 정리
4. templates/card-news/news.md 템플릿에 채우기
```

### Phase 2: 반자동 (나중에)
```
- Perplexity / Claude 웹검색으로 "이번 주 방탈출 신규 오픈" 주간 집계 프롬프트
- 결과를 사람이 검증 → 카드뉴스화
```

### Phase 3: 자동 (개발자 영역, 추후)
```
- 매장 공홈/예약페이지 크롤링 + RSS + 인스타 모니터링
- 신규 감지 → AI 요약 → 카드뉴스 자동 생성 → 검토 후 발행
- 자세한 설계는 pipeline/ 에서
```

> ⚠️ 소식은 **사실 정확성**이 생명. 오픈일·가격·위치는 반드시 출처 확인. AI 집계 결과를 그대로 믿지 말 것.

## 6. 배치 생산 워크플로우

영감/시간 있는 날 한 번에:

```
[생산 세션 — 2~3시간]
1. content/ideas/BACKLOG.md 에서 만들 주제 3~5개 선택
2. 각 주제 → content/drafts/ 에 템플릿 복사 후 텍스트만 채우기 (카피)
   (디자인은 아직 X, 텍스트만 — "글쓰기 모드" 한 번에 몰아서)
3. 카피 다 되면 → 미리캔버스에서 한 번에 디자인 ("디자인 모드" 몰아서)
4. 완성 PNG 저장 + frontmatter status: ready 로 표시 (= 재고)
```

> **핵심**: "글쓰기 모드"와 "디자인 모드"를 섞지 말 것. 뇌 전환 비용이 큼.

## 7. 발행 워크플로우 (발행일에)

```
[발행 — 10분]
1. content/drafts/ 에서 status: ready 중 캘린더에 맞는 카테고리 선택
2. PNG + 캡션을 Meta Business Suite(또는 Buffer)에 업로드
3. 캘린더 시간에 예약 (또는 즉시 게시)
4. 파일을 content/published/ 로 이동 + git commit
```

> 더 나아가면: 2주치를 한 번에 예약 게시 걸어두기 → 발행 자체를 자동화.

## 8. 상태(status) 라이프사이클

frontmatter의 `status` 필드로 관리:
```
idea     → 백로그에만 있는 주제 (아직 안 만듦)
draft    → 카피 작성 중
ready    → 카피+디자인 완성, 발행 대기 (= 재고)
scheduled→ 예약 게시 걸어둠
published→ 게시 완료
```

재고 = `status: ready` 개수. 4개 이상이면 안심. [BACKLOG.md](../content/ideas/BACKLOG.md) 상단에 표시.
