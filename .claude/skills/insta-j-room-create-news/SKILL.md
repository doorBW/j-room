# insta-j-room-create-news — 소식 카드뉴스 제작

> j-room 인스타그램의 **소식(news)** 정기 코너. 주간 신규 오픈/테마/이벤트/예약 소식.
> 레퍼런스: `pipeline/data/news-sample.json` (레이아웃 데모)

---

## 0. 핵심 원칙 — 소식은 다르다

- **시의성**: 미리 만들면 낡는다 → **발행 직전(그 주에) 집계**. 재고로 쌓아두지 말 것.
- **정확성이 생명**: 오픈일/가격/위치가 틀리면 신뢰 추락 → **출처 교차 확인 필수**.
- **AI는 보조**: 일반 웹검색은 신규 오픈 실시간 정보가 약함. 매장 공식 채널을 직접 확인하고 사람이 팩트체크.
- 발행 슬롯: 격주 **2주차 수요일** (CONTENT_STRATEGY 기준).

---

## 1. 카드 구조 (가변 6~8장)

| # | type | 역할 |
|---|------|------|
| 1 | `cover` | 표지 — "이번 주 방탈출 소식" + NEWS #회차 + 주차 |
| 2 | `intro` | 이번 주 한눈에 (요약 불릿 3~5) |
| 3~N | `newsitem` | 소식 1건 = 1장 (보통 3~5건) |
| N+1 | `cta` | 가보고 싶은 곳 댓글 + 다음 주 예고 |

> **카드2 주의**: 소식은 추천과 달리 카드2가 "쫄보 4인방 자기소개"가 **아니라** "이번 주 한눈에(요약)". 쫄보 브랜딩은 표지 kicker("쫄보 4인방이 모아온")로 녹인다.

---

## 2. data JSON 스키마

`meta` + `cards[]` (추천과 동일 골격, cover/intro/cta 재사용).

### cover
```json
{ "type": "cover", "kicker": "쫄보 4인방이 모아온", "title": "이번 주 방탈출 소식", "titleAccent": "NEWS #N", "hook": "신상 떴다, 예약 서둘러 🗞️", "sub": "YYYY년 M월 W주차 · 신규 오픈 & 예약 소식" }
```

### intro (이번 주 한눈에)
```json
{ "type": "intro", "title": "이번 주 한눈에", "subtitle": "출처는 각 매장 공식 채널 ✔️", "points": ["🆕 신규 오픈 <span class=\"accent\">3곳</span>", "🎉 이벤트 1건", "🔥 예약 전쟁 예상 1개"] }
```

### newsitem (소식 1건)
```json
{
  "type": "newsitem",
  "tag": "🆕 신규 오픈",         // 🔁 리뉴얼 / 🎉 이벤트 / 🔥 예약 오픈 등
  "poster": "assets/images/<매장>/<테마>.jpg",  // 선택 (있으면 정사각 표시)
  "store": "매장명",
  "name": "테마명",
  "info": [                        // 칩으로 표시. 필요한 것만
    { "label": "📅 오픈", "value": "6/20" },
    { "label": "📍", "value": "강남" },
    { "label": "🎭", "value": "공포·스릴러" },
    { "label": "👥", "value": "2~4인" },
    { "label": "💰", "value": "오픈 할인 20%" }
  ],
  "comment": "한 줄 코멘트 (기대 포인트 등)",
  "source": "@매장_공식계정"       // 출처 필수
}
```

### cta
```json
{ "type": "cta", "title": "가보고 싶은 곳 있어요?", "actions": ["댓글로 찜 남기기 👇", "저장해두고 예약 알림 💾"], "next": "다음 주 소식도 쫄보 4인방이 모아올게요 🗞️" }
```

## 3. 집계 워크플로우 (발행 직전)

### STEP 1 — 소스 수집 (주 1회, ~30분)
- 매장 **공식 인스타 / 공홈 / 예약 페이지** (가장 정확)
- 방탈출 커뮤니티 (네이버 카페, 디스코드)
- 해시태그: #방탈출신규 #방탈출오픈 #방탈출신상
- 예약 플랫폼 신규 등록 (예약통 등)
- 사용자(쫄보 4인방)가 직접 아는 소식

### STEP 2 — 추리기 (3~5건)
- 신규 오픈 / 신규 테마 / 리뉴얼 / 이벤트·할인 / 예약 오픈 임박
- 지역·장르 다양하게 섞기

### STEP 3 — 팩트체크 ⚠️ (가장 중요)
각 소식의 **오픈일·가격·위치·매장명·테마명**을 출처에서 교차 확인.
- AI 검색 결과를 그대로 쓰지 말 것
- 불확실하면 빼거나 "예정" 표기
- 포스터 쓰면 출처 표기 + SOURCES.md 기록 (저작권)

### STEP 4 — data JSON + 렌더
```bash
cd pipeline
# data/news-YYYY-MM-wN.json 작성
node render.js data/news-YYYY-MM-wN.json
```

### STEP 5 — 검토 + 발행
- 오타·날짜·가격 재확인, 출처 표기 확인
- content/drafts → published, git commit

## 4. 정확성 체크리스트

- [ ] 모든 오픈일/가격/위치를 출처에서 교차 확인했는가
- [ ] AI 집계를 그대로 쓰지 않고 매장 공식 채널로 검증했는가
- [ ] 매장명·테마명 오타 없는가
- [ ] 출처(@계정)를 각 소식·표지에 표기했는가
- [ ] 시의성 OK (이번 주 집계, 낡은 정보 아님)

---

## 5. 자동화 로드맵 (개발자용, 추후)

- **Phase 1 (현재)**: 수동 집계 + AI 검색 보조
- **Phase 2**: Perplexity/Claude 웹검색 주간 집계 프롬프트 → 사람 검증
- **Phase 3**: 매장 공홈/예약/인스타 크롤링 + 신규 감지 → AI 요약 → 카드 자동 생성 → 검토 후 발행
- 렌더러: `pipeline/lib/cards.js` 의 `newsitem` 타입, 스타일은 `styles.css` 의 `.news__*`
